Recommendations

Design improvements and actionable suggestions to enhance the usability, accessibility, and overall user experience of MediQuick.

Key Improvements

Top Recommendations

High-impact usability improvements derived from the heuristic evaluation and usability testing results, organized by priority.

🛡️
Implement Checkout Validation
Add real-time inline form validation with clear required field indicators and a full order review screen before final submission to prevent user errors and failed orders.
💬
Rewrite Error Messages
Replace all generic system errors with specific, actionable, human-friendly messages that explain what went wrong and provide clear steps for the user to recover.
🎨
Establish a Design System
Create a unified component library with consistent button styles, icon usage, typography scales, and color tokens to ensure visual consistency across all screens.
🏷️
Simplify Medicine Categories
Replace technical medical labels with everyday language (e.g., "Pain Relief" instead of "Analgesics") and add descriptive subtitles to help users find what they need quickly.
📄
Redesign Product Pages
Restructure product detail pages with a clear information hierarchy — lead with the product name, key details, and a prominent "Add to Cart" button, moving secondary details below.
🕐
Add Recently Viewed
Implement a "Recently Viewed" section and persistent search history to reduce memory load and help users return to products they have previously browsed.
Implementation Roadmap

Suggested Design Improvements

Recommendations organized by priority and suggested timeline for the next iteration of the MediQuick prototype.

🔴 Immediate Priority
Form Validation & Error Prevention System
Implement complete form validation across all input fields in the checkout flow. Add required field indicators (*), inline error messages that appear before submission, and a final order confirmation/review screen that allows users to check their details before placing the order. This directly addresses the highest-severity heuristic violation found in the evaluation.
🔴 Immediate Priority
Error Message Overhaul
Audit every error state in the prototype and replace all generic messages ("Something went wrong", "Error 404") with specific, human-readable descriptions. Each error message should clearly state: what happened, why it happened, and what the user should do next. Example: "This item is currently out of stock. Try browsing similar products below."
🟡 Short-Term (Next Sprint)
UI Consistency & Design System
Define and document a core design system for MediQuick: establish a primary button style (filled teal), secondary button style (outlined), icon set (single library), and typography scale. Apply these consistently across all screens. This will reduce cognitive friction and increase the system's perceived professionalism.
🟡 Short-Term (Next Sprint)
Medicine Category Labels & Navigation
Rename all medicine categories to plain language alternatives. Add short descriptive text beneath each category tile on the home screen. Consider adding a "What are you looking for?" guided search for first-time users who may not know the medical name of their medicine.
🟢 Long-Term Enhancement
Product Page Information Architecture
Conduct a card-sorting exercise with users to determine the ideal order of information on product pages. Implement a collapsible "Details" section for secondary information (ingredients, warnings, storage instructions), keeping the primary view clean with just the product name, image, price, and Add to Cart button.
🟢 Long-Term Enhancement
Recently Viewed & Order History Features
Develop a "Recently Viewed" shelf on the home screen and product listing pages. Add a one-click "Reorder" feature in the order history section for returning customers. These features leverage recognition over recall and significantly improve efficiency for repeat users.

Suggested Design Changes

Specific before-and-after examples illustrating how identified issues can be resolved in the next prototype iteration.

Error Messages

Category Labels

Category names should use language that everyday users — not medical professionals — can immediately understand.

Before
"Analgesics / Antipyretics / Antihypertensives"
After
"Pain Relief / Fever Care / Blood Pressure"

Checkout Flow

Add a confirmation step before final order submission to give users a chance to review and correct their details.

Before
Fill form → Place Order (no review)
After
Fill form → Review Order → Confirm → Place Order

Product Page Layout

Reduce visual clutter on product pages by separating essential information from secondary details.

Before
All info shown at once, CTA buried in content
After
Name + Price + CTA above fold, details collapsible below

Conclusion

Final Remarks

The MediQuick prototype demonstrates strong potential as a user-friendly online pharmacy platform. With an SUS score of 77.5, the system is above the industry benchmark, indicating that users found it generally acceptable and easy to learn. By addressing the critical issues identified — particularly around error prevention and error messaging — and implementing the recommended design improvements, MediQuick can achieve an even higher standard of usability and better serve the needs of its target users.

This evaluation was conducted as part of the HCI2 Final Project at Partido State University under the supervision of Michael B. Amata, Course Instructor, Computational Sciences Department.
